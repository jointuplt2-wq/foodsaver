## 1. 프로젝트 개요

- **프로젝트명:** FoodSaver
- **목표:** 식품 소비기한·재고 수량을 브라우저 로컬 스토리지만으로 추적하여 폐기 손실 최소화

## 2. 기능 명세

### 2.1 대시보드

| 항목 | 상세 |
|------|------|
| 총 재고 품목 수 | 전체 등록 항목 카운트 |
| 위험 항목 수 | 7일 이내 만료 품목 (빨간 뱃지) |
| 주의 항목 수 | 8~30일 이내 만료 품목 (노란 뱃지) |
| 만료 항목 수 | 소비기한 경과 품목 (회색·취소선) |
| 오늘 날짜 표시 | 기준일 명시로 계산 혼동 방지 |

> **PLAN.md 수정:** 경고 기준이 "30일 이내" 단일 단계였으나, 7일·30일 2단계로 세분화하여 우선순위 판단 강화.

---

### 2.2 식품 등록 폼

필수 필드:

| 필드 | 타입 | 비고 |
|------|------|------|
| 식품명 | 텍스트 | |
| 카테고리 | 드롭다운 | 냉장 / 냉동 / 상온 / 가공 / 신선 |
| 입고일 | 날짜 | 기본값: 오늘 |
| 소비기한 | 날짜 | 입고일 이후여야 함 (유효성 검사) |
| 수량 | 숫자 | 최소 1 |
| 단위 | 드롭다운 | 개 / g / mL / kg / L |
| 보관 장소 | 텍스트 | 예: 냉장고 2단, 창고 A |
| 최소 재고 임계값 | 숫자 (선택) | 이 수량 미만 시 '재고 부족' 경고 |

> **PLAN.md 보완:** 단위 필드·최소 재고 임계값·입고일 기본값·유효성 검사가 누락되어 추가.

---

### 2.3 재고 리스트

- **정렬:** 소비기한 오름차순 (가장 임박한 항목이 최상단)
- **검색:** 식품명·보관 장소 실시간 필터
- **카테고리 탭 필터:** 전체 / 냉장 / 냉동 / 상온 / 가공 / 신선
- **수량 조작:** `[+]` `[-]` 버튼, 수량 0 도달 시 "소진 완료" 확인 팝업 후 자동 삭제
- **삭제 흐름 구분:**
  - 소진(수량 0): 자동 제거 — 정상 소진으로 기록
  - 폐기(만료·강제 삭제): 삭제 버튼 → 확인 팝업 후 제거

> **PLAN.md 명확화:** "삭제 기능" 설명이 소진·폐기를 구분하지 않아 UX 혼란 가능성 있었음.

---

### 2.4 스마트 경고 시스템

| 상태 | 조건 | 시각 표시 |
|------|------|-----------|
| 만료 | D-day < 0 | 회색 배경, 취소선, "만료" 뱃지 |
| 위험 | 0 ≤ D-day ≤ 7 | 빨간 배경, 깜박임 펄스 |
| 주의 | 8 ≤ D-day ≤ 30 | 노란 배경 |
| 정상 | D-day > 30 | 기본 배경 |
| 재고 부족 | 수량 < 최소 임계값 | 파란 테두리 경고 |

---

### 2.5 데이터 내보내기 / 가져오기 (신규)

- **CSV 내보내기:** 전체 재고를 CSV 파일로 다운로드 (Excel 호환)
- **CSV 가져오기:** CSV 업로드 시 기존 데이터에 병합 (중복 식품명·소비기한 동일 시 수량 합산)
- **JSON 백업:** 로컬 스토리지 전체를 JSON으로 백업/복원 (브라우저 이전 대비)

## 3. 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 마크업 | HTML5 | 단일 파일 배포 |
| 스타일 | CSS3 (Glassmorphism) | 별도 라이브러리 불필요 |
| 스크립트 | Vanilla JavaScript (ES6+) | 의존성 없음, 오프라인 완전 동작 |
| 저장소 | localStorage | 서버 불필요, 영구 보존 |
| 아이콘 | Font Awesome CDN | 소량 사용, CDN 1줄 |
| 날짜 처리 | 내장 Date API | 외부 라이브러리 없이 충분 |

> **PLAN.md 보완:** 기술 스택이 전혀 명시되지 않았음.

## 6. UX 원칙

1. **로딩 없음** — 모든 동작이 클라이언트 측에서 즉시 반응
2. **입력 최소화** — 입고일 기본값 오늘, 단위 기본값 "개"
3. **파괴적 동작 확인** — 삭제·소진 완료 시 반드시 확인 팝업
4. **모바일 대응** — 768px 이하 반응형 레이아웃 (테이블 → 카드 뷰)
